- 用户持续要求前端对齐参考站时，优先直接删掉多余文案和装饰，不要保留“解释型”标题、副标题、状态标签。
- queue 区优先做成轻列表，不要保留厚卡片、缩略图和多余操作；状态胶囊要尽量贴参考实现。
- drop 区优先让输入框和按钮成为唯一主角，按钮并排右对齐，说明文案尽量并入输入区域内部。
- queue 头部和行内操作要继续收紧：标题、图标、计数胶囊、clear、下载/展开按钮都按参考站的小尺寸纯图标方案处理。
- 右侧设置区不要重复放主操作按钮，能并到上层操作区的就并掉，只保留必要设置和清理入口。
- 当用户要求把平台/自动选择收进配置区时，头部继续只保留标题和必要胶囊，不要把筛选胶囊留在顶栏。
- 配置区只保留设置本身，不保留 ready/selected/img-vid-aud 这类统计文案；主操作按钮紧贴相关设置块摆放。
- 头部标题和胶囊按钮要单独抽成复用控件样式，尺寸、间距、圆角、位置统一，不要每个头部各写一套。
- 整体布局优先用全局变量统一壳宽、主网格 gap、卡片 padding、区块间距，避免每个模块各写一套数值导致漂移。
- 参考站对齐时，Queue 和 Configuration 头部优先抄“外置轻标题 + 小图标 + 小字号大写灰字”，不要把标题塞回厚卡片内部。
- drop 区对齐参考站时，优先收框体本身：白底实线浅边框、紧 padding、浅灰标题和 placeholder，先压气质再动功能。
- drop 区若出现“多层框感”，优先删输入框那层边框，保留外层主框；按钮必须同尺寸并排，固定在右下角收口。
- drop 外面如果还有包裹卡片边框，直接把外层 workspace 容器去框去阴影，只保留 drop 自己这一层主框。
- drop 下方信息卡和 queue 空状态都要走“轻卡 + 浅灰字 + 小字号”路线，避免大数字和重卡片感抢主界面。
- drop 下方信息卡若保留数字，优先把数字放到右侧做次强调，左侧只留标签和说明，避免卡片中心被大数字占满。
- 头部主标题、语言胶囊、CONFIGURATION 头部要一起收敛：优先缩距离、缩高度、缩字距，保持一条干净基线。
- 三张信息卡要统一高度和内边距；CONFIGURATION 头部要用微小 padding 调基线，不靠大 margin 硬顶。
- 如果用户要求主标题跟 drop 左对齐，优先给品牌区加和 drop 相同的左内边距，让标题对齐 drop 内容起点。
- 操作区里“功能标题 + 功能选择按钮区”要看成一个组件：标题高度、按钮容器高度、块 padding 统一；主区域圆角统一走全局 radius 变量。
